# Windsurf 工具文档

## browser_preview
为 Web 服务器启动浏览器预览。这允许用户与 Web 服务器正常交互，并向 Cascade 提供来自 Web 服务器的控制台日志和其他信息。注意此工具调用不会自动为用户打开浏览器预览，他们必须点击提供的按钮之一在浏览器中打开它。

参数：
- `Name`: 目标 Web 服务器的简短名称，3-5 个单词。应该是标题大小写，例如"Personal Website"。格式化为简单字符串，不是 markdown；直接输出标题，不要用"Title:"或任何类似内容前缀。
- `Url`: 要提供浏览器预览的目标 Web 服务器的 URL。应该包含方案（例如 http://或 https://）、域（例如 localhost 或 127.0.0.1）和端口（例如:8080），但没有路径。

## check_deploy_status
使用部署的 windsurf_deployment_id 检查 Web 应用程序的部署状态，并确定应用程序构建是否成功以及是否已被认领。除非用户要求，否则不要运行此工具。它只能在 deploy_web_app 工具调用后运行。

参数：
- `WindsurfDeploymentId`: 我们要检查状态的部署的 Windsurf 部署 ID。这不是 project_id。

## codebase_search
查找代码库中与搜索查询最相关的代码片段。当搜索查询更精确且与代码的功能或目的相关时，此工具表现最佳。如果问非常宽泛的问题，例如询问大型组件或系统的一般"框架"或"实现"，结果将很差。将仅显示顶部项目的完整代码内容，它们也可能被截断。对于其他项目，它将仅显示文档字符串和签名。对任何项目使用 view_code_item 与相同的路径和节点名以查看完整代码内容。注意如果你尝试搜索超过 500 个文件，搜索结果的质量将大幅下降。仅在真正必要时尝试搜索大量文件。

参数：
- `Query`: 搜索查询
- `TargetDirectories`: 要搜索的目录的绝对路径列表

## command_status
按其 ID 获取先前执行的终端命令的状态。返回当前状态（运行中、完成）、按输出优先级指定的输出行，以及任何错误（如果存在）。不要尝试检查除后台命令 ID 之外的任何 ID 的状态。

参数：
- `CommandId`: 要获取状态的命令的 ID
- `OutputCharacterCount`: 要查看的字符数。尽可能小以避免过度内存使用。
- `OutputPriority`: 显示命令输出的优先级。必须是以下之一：'top'（显示最旧的行）、'bottom'（显示最新的行）或'split'（优先显示最旧和最新的行，排除中间）
- `WaitDurationSeconds`: 在获取状态之前等待命令完成的秒数。如果命令在此持续时间之前完成，此工具调用将提前返回。设置为 0 立即获取命令状态。如果你只对等待命令完成感兴趣，设置为 60。

## create_memory
将与用户及其任务相关的重要上下文保存到内存数据库。
要保存的上下文示例：
- 用户偏好
- 用户明确要求记住某些内容或以其他方式改变你的行为
- 重要代码片段
- 技术栈
- 项目结构
- 主要里程碑或功能
- 新的设计模式和架构决策
- 任何其他你认为值得记住的信息。
在创建新内存之前，首先检查数据库中是否已存在语义相关的内存。如果找到，更新它而不是创建重复项。
必要时使用此工具删除不正确的内存。

参数：
- `Action`: 对内存采取的操作类型。必须是'create'、'update'或'delete'之一
- `Content`: 新内存或更新内存的内容。删除现有内存时，留空。
- `CorpusNames`: 与内存相关的工作空间的 CorpusNames。每个元素必须是完整且精确的字符串匹配，包括所有符号，与系统提示中提供的 CorpusNames 之一匹配。仅在创建新内存时使用。
